# [PRD] 전화번호 관리 시스템 (Phonebook Management System)

## 1. 개요 (Product Overview)
본 프로젝트는 사용자가 개인의 연락처 리스트를 쉽고 빠르게 관리할 수 있는 웹 기반의 '전화번호 관리 시스템'입니다. 보안과 성능을 최우선으로 하며, 최신 웹 기술 스택을 활용하여 확장성과 유지보수성을 확보합니다.

## 2. 목표 (Goals)
- **사용자 편의성**: 직관적인 인터페이스를 통해 연락처와 **분류(카테고리)**의 추가, 수정, 삭제, 검색을 간편하게 수행.
- **보안 및 안정성**: Supabase의 보안 기능을 활용하며, **이름 및 전화번호와 같은 민감한 정보는 암호화하여 저장**함으로 데이터 보안을 강화.
- **인스타그램 스타일 UI/UX**: 깔끔하고 현대적인 인스타그램 감성의 디자인 적용 (미니멀리즘, 아이콘 중심, 원형 아바타 등).
- **반응형 디자인**: 모바일 및 데스크탑 환경 모두에서 최적화된 사용자 경험 제공.

## 3. 핵심 기능 (Core Features)
### 3.1 연락처 관리
- **연락처 추가**: 이름, 전화번호, 분류(카테고리), 메모를 입력하여 새로운 연락처를 생성. (이름, 전화번호는 암호화 처리)
- **연락처 목록**: 저장된 전체 연락처를 리스트 형태로 조회. (분류별 필터링 지원)
- **연락처 수정**: 기존 연락처의 정보를 업데이트. (분류 변경 포함)
- **연락처 삭제**: 불필요한 연락처를 영구 삭제.

### 3.2 분류(카테고리) 관리
- **분류 추가**: 새로운 분류(예: 친구, 친척, 동아리 등)를 생성.
- **분류 수정/삭제**: 기존 분류의 이름을 수정하거나 삭제.
- **분류별 조회**: 특정 분류에 속한 연락처만 필터링하여 조회.

### 3.2 검색 기능
- **부분 일치 검색**: 이름 또는 전화번호의 일부만 입력해도 실시간으로 검색 결과 필터링. (데이터가 암호화되어 있어, 클라이언트 사이드 또는 서버 사이드 복호화 후 검색 수행)

## 4. 기술 스택 (Technical Stack)
- **Frontend**: Next.js (App Router), Tailwind CSS
- **Backend/Database**: Supabase (Postgres)
- **Deployment**: Vercel
- **State Management**: React Hooks (useState, useEffect) or React Query (추후 고려)

## 5. 데이터 스키마 (Data Schema)
### `categories` 테이블 (신규)
| 컬럼명 | 타입 | 설명 | 제약 조건 |
| :--- | :--- | :--- | :--- |
| `id` | UUID | 고유 식별자 | PK |
| `name` | Text | 분류명 (예: 친구) | Not Null, Unique |
| `created_at` | Timestamp | 생성 시점 | Default: now() |

### `contacts` 테이블 (수정)
| 컬럼명 | 타입 | 설명 | 제약 조건 |
| :--- | :--- | :--- | :--- |
| `id` | UUID | 고유 식별자 | PK |
| `name` | Text | 이름 (암호화) | Not Null |
| `phone` | Text | 전화번호 (암호화) | Not Null |
| `category_id` | UUID | 분류 식별자 | FK (categories.id), Nullable |
| `memo` | Text | 메모 | Optional |
| `created_at` | Timestamp | 생성 시점 | Default: now() |
| `updated_at` | Timestamp | 수정 시점 | Default: now() |

## 6. 사용자 흐름 및 UI/UX 요구사항
- **메인 페이지**: 인스타그램 피드 스타일의 연락처 리스트. 상단에는 분류(카테고리)가 '스토리' 스타일의 가로 스크롤 위젯으로 배치.
- **연락처 카드**: 원형 프로필 이미지(이름 이니셜)와 깔끔한 텍스트 배치. 우측에 액션 버튼(수정, 삭제)을 아이콘으로 간결하게 표현.
- **입력 폼**: 하단 시트(Bottom Sheet) 또는 모달 형태의 깔끔한 입력창.
- **반응형 레이아웃**: 모바일에서는 하단 탭 내비게이션, 데스크탑에서는 상단/측면 내비게이션 제공.
- **디자인 테마**: 화이트 배경(라이트 모드)과 블랙 포인트, 부드러운 라운드 처리된 카드 디자인.

## 7. 보안 및 성능 (Security & Performance)
- **보안 (Security)**:
  - **데이터 암호화**: AES-256-GCM 알고리즘 등을 사용하여 이름(name)과 전화번호(phone)를 암호화하여 저장.
  - Supabase RLS (Row Level Security)를 설정하여 권한이 없는 사용자의 데이터 접근 차단.
  - Server-side API Route 또는 Server Actions를 통한 데이터 처리 권장.
- **성능 (Performance)**:
  - Next.js의 SSR 및 ISR을 활용하여 초기 로딩 속도 최적화.
  - 이미지 및 정정 자산은 Vercel 에지망을 통해 서빙.
